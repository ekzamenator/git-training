# Памятка по Git

## Проверка состояния репозитория
git status
Показывает:
- измененные файлы;
- файлы в staging;
- неотслеживаемые файлы;
- состояние относительно origin.

## Просмотр истории
git log
Подробная история коммитов.

git log --oneline
Краткая история.

git log --oneline --decorate --graph --all
История с ветками и указателями.

## Просмотр изменений
git diff
Показывает изменения в рабочей папке относительно последнего коммита.

## Добавление файлов в staging
git add <файл>
Добавить один файл в staging.

git add .
Добавить все изменения в текущей папке и вложенных каталогах.

## Создание коммита
git commit -m "Сообщение"
Создать коммит с сообщением.

git commit
Открыть редактор для ввода одно- или многострочного сообщения.

## Отмена изменений в рабочей папке
git restore <файл>
Вернуть файл к состоянию последнего коммита.

git restore .
Отменить все незакоммиченные изменения.

## Убрать файл из staging
git restore --staged <файл>
Убрать файл из staging, оставив изменения в рабочей папке.

## Работа с удаленным репозиторием
git remote -v
Показать удаленные репозитории.

git push
Отправить коммиты на GitHub.

git push -u origin main
Связать локальную ветку main с origin/main и отправить изменения.

git pull
Получить изменения и выполнить merge.

## Важные понятия

Рабочая папка (Working Directory)
↓
Staging (Index)
↓
Коммит (Repository)

git add      : Working Directory -> Staging
git commit   : Staging -> Repository
git restore  : Repository -> Working Directory
git push     : Repository -> GitHub
git pull     : GitHub -> Repository

Типичный цикл работы
Изменил файлы
      ↓
git status
      ↓
git diff
      ↓
git add .
      ↓
git commit -m "Что сделал"
      ↓
git push

git status показывает состояние рабочей директории и индекса.
