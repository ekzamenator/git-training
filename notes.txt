# Памятка по Git

## Проверка состояния
git status
Показать:
- измененные файлы;
- файлы в staging;
- неотслеживаемые файлы;
- состояние относительно удаленного репозитория.

## История
git log
Подробная история коммитов.

git log --oneline
Краткая история.

git log --oneline --graph --decorate --all
История в виде графа веток.

git show
Информация о коммите и его изменения.

git reflog
История перемещений HEAD (часто помогает восстановить потерянные коммиты).

## Просмотр изменений
git diff
Изменения рабочей директории относительно staging.

git diff --staged
Изменения в staging относительно последнего коммита.

## Добавление в staging
git add <файл>
Добавить один файл.

git add .
Добавить все изменения.

## Создание коммитов
git commit -m "Сообщение"
Создать коммит с сообщением.

git commit
Открыть редактор для ввода сообщения.

## Отмена изменений в рабочей директории
git restore <файл>
Вернуть файл к состоянию последнего коммита.

git restore .
Отменить все незакоммиченные изменения.

## Работа со staging
git restore --staged <файл>
Убрать файл из staging, оставив изменения в рабочей директории.

git restore --staged .
Очистить весь staging.

## Работа с ветками
git branch
Показать ветки.

git branch <ветка>
Создать ветку.

git switch <ветка>
Переключиться на ветку.

git switch -c <ветка>
Создать ветку и сразу переключиться на нее.

## Слияние веток
git merge <ветка>
Влить указанную ветку в текущую.

Fast-forward merge:
main просто перемещается вперед, merge-коммит не создается.

Merge commit:
создается отдельный коммит с двумя родителями.

## Работа с удаленным репозиторием
git remote -v
Показать удаленные репозитории.

git push
Отправить коммиты на GitHub.

git push -u origin <ветка>
Опубликовать ветку и связать ее с удаленной.

git pull
Получить и сразу применить изменения.

git fetch
Получить изменения без применения.

## Откат истории
git revert <коммит>
Создать новый коммит, отменяющий изменения указанного коммита.
История не переписывается.

git reset --hard <коммит>
Переместить ветку и рабочую директорию к указанному коммиту.
История переписывается.

git reset --hard HEAD~1
Удалить последний коммит локально.

## Восстановление после reset
git reflog
Найти потерянный коммит.

git reset --hard <hash>
Вернуться к потерянному коммиту.

--------------------------------------------------

Три состояния Git:

Рабочая директория (Working Directory)
        ↓
Staging (Index)
        ↓
Коммит (Repository)
        ↓
GitHub (Remote Repository)

git add      : Working Directory -> Staging
git commit   : Staging -> Repository
git push     : Repository -> Remote
git pull     : Remote -> Repository + Working Directory
git restore  : Repository -> Working Directory
